## 68. บทนี้สำคัญเมื่อไร

เมื่อคุณสร้างเหล็กในโมเดลได้แล้ว ขั้นถัดไปที่สำคัญคือการเปลี่ยนโมเดลให้เป็นเอกสารใช้งานจริง เช่น

- รายละเอียด
- ตารางเหล็ก
- รายงาน

บทนี้จึงเน้นเครื่องมือกลุ่มปลายทาง

- `Detail Rebar`
- `Reinforcement Schedule`
- `Generate Report`

### สิ่งที่ต้องพร้อมก่อนเริ่มบทนี้

1. โมเดลเหล็กต้องนิ่งพอแล้ว
2. เช็กผลใน `3D` และ `Section` แล้ว
3. bar mark ควรเริ่มมีหลักการที่ชัด
4. ถ้าจะใช้บทบาทของเหล็กในเอกสาร ให้กลับไปอ่านบท `Assign Rebar Roles และ Clone Drawings` ก่อน

> [!WARNING]
> ถ้าโมเดลยังเป็นของทดลอง หรือยังมี warning สำคัญค้างอยู่ อย่าเพิ่งทำ `Detail Rebar`, `Reinforcement Schedule`, หรือ `Generate Report` เพราะข้อมูลปลายทางจะผิดตามโมเดลไปด้วย

---

## 69. Detail Rebar

ใช้สร้างรายละเอียดของเหล็กจากโมเดล เช่น

- bending detail
- annotation
- dimension
- ข้อมูลประกอบสำหรับแบบ

เหมาะกับงานที่โมเดลถูกแล้ว และต้องการสื่อสารต่อในแบบ

### วิธีใช้แบบเริ่มต้น

1. เปิด view ที่ต้องการทำ detail
2. เลือก host หรือชุดเหล็กที่ต้องการอธิบาย
3. ไปที่แท็บ `PowerPack Rebar`
4. คลิก `Detail Rebar`
5. เลือกชนิด detail ที่ต้องการ
6. กด `OK` หรือ `Apply` ตามหน้าต่างที่ใช้
7. ตรวจว่า annotation และสัญลักษณ์ไม่ลอยผิดตำแหน่ง

### Checkpoint ก่อนขยับไป Schedule

1. detail ที่ได้อ้างอิงเหล็กชุดที่ถูกต้อง
2. annotation ไม่ลอยหลุดจาก host
3. ยังไม่มีเส้นเหล็กทดลองหรือ detail เก่าซ้อนกันใน view

## 70. Reinforcement Schedule

ใช้สร้างตารางเหล็กจากข้อมูลในโมเดล

ก่อนใช้คำสั่งนี้ ควรเช็กว่า

1. bar mark หรือหมายเลขเหล็กถูกต้อง
2. บทบาทของเหล็กถูกต้องพอสมควร
3. โมเดลไม่มีเหล็กทดลองที่ไม่ใช้จริงค้างอยู่

### วิธีใช้แบบเริ่มต้น

1. ไปที่แท็บ `PowerPack Rebar`
2. คลิก `Reinforcement Schedule`
3. เลือก template หรือรูปแบบตารางที่ต้องการ
4. กดสร้างตาราง
5. เปิด schedule ที่ได้แล้วเช็ก bar mark, quantity และ type

### Checkpoint หลังสร้าง Schedule

1. จำนวน bar mark ไม่มากผิดปกติ
2. ไม่มีชนิดเหล็กทดลองค้างอยู่ในตาราง
3. quantity ไม่เพิ่มเพราะเผลอ copy หรือ create ซ้ำ

## 71. Generate Report

ใช้สร้างรายงานจากข้อมูลผลลัพธ์หรือการทำงานของ element

เหมาะกับการสรุปผลลัพธ์และส่งต่อเอกสารภายในทีม

### วิธีใช้แบบเริ่มต้น

1. เลือก element หรือชุดงานที่ต้องการสรุป
2. คลิก `Generate Report`
3. เลือกรูปแบบรายงาน
4. สร้างและเปิดดูผลลัพธ์
5. เช็กว่าข้อมูลในรายงานตรงกับโมเดลล่าสุด

### Checkpoint ก่อนส่งออก

1. report ดึงจาก element ที่ถูกต้อง
2. ข้อมูลล่าสุดตรงกับ `3D` และ `Section`
3. ไม่มี warning สำคัญค้างอยู่ใน host ที่เกี่ยวข้อง

---

## 72. ลำดับปลายทางที่แนะนำ

1. ทำโมเดลเหล็กให้ถูกก่อน
2. ตั้งหมายเลขเหล็กให้สื่อความหมาย
3. ใช้ `Detail Rebar`
4. ใช้ `Reinforcement Schedule`
5. ใช้ `Generate Report`

---

## 73. ข้อควรระวัง

- อย่ารีบทำ schedule ตอนโมเดลยังไม่เสถียร
- อย่าเริ่ม detail ก่อนตรวจ `3D` และ `Section`
- ถ้า bar mark ยังไม่นิ่ง ตารางเหล็กจะเปลี่ยนตามไปด้วย
