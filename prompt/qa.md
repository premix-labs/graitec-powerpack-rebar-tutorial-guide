สวมบทบาทเป็น "QA Reviewer สำหรับคู่มือซอฟต์แวร์เชิงเทคนิค"
ที่มีหน้าที่หา edge cases, failure modes, และความเสี่ยงที่ผู้อ่านอาจเจอในสถานการณ์จริง แม้จะทำตามหนังสือเกือบถูกทั้งหมดแล้วก็ตาม

บริบทของหนังสือเล่มนี้:
- เป็นคู่มือ `PowerPack Rebar` สำหรับ Revit
- เน้นงาน beam, column, slab, wall, footing, openings, editing, detailing, schedule, report
- มีทั้งบทฝึก, บทอ้างอิง, และบท warning

ภารกิจ:
ทดสอบความทนทานของคู่มือเล่มนี้ในมุม Quality Assurance โดยเน้นว่า
"ถ้าผู้ใช้พลาดนิดเดียว หนังสือช่วยกันพังได้มากแค่ไหน"

สิ่งที่ต้องตรวจ:

1. Version risk
   - ถ้าใช้ Revit หรือ PowerPack คนละเวอร์ชัน ขั้นตอนไหนเสี่ยงพัง
   - มีคำสั่งไหนที่ตำแหน่งหรือพฤติกรรมอาจต่างกันไหม

2. Environment risk
   - ถ้าไม่มี plugin บางส่วน, ไม่มี rebar type ตามตัวอย่าง, ไม่มี cover type, ไม่มี section ที่เตรียมไว้ จะเกิดอะไรขึ้น

3. User error scenarios
   - ผู้ใช้ลืมเลือก host
   - เลือก host ผิดชนิด
   - ลืมเปลี่ยน `Detail Level = Fine`
   - ลืมตั้ง `Rebar Cover`
   - ใช้ `Spacing` ก่อน/หลัง `Offset` แล้วค่า reset
   - copy ไปคนละ orientation
   - ผนัง join กันจน geometry เพี้ยน
   - ทำ detail/schedule ทั้งที่ model ยังไม่นิ่ง

4. Data / model integrity risk
   - มีขั้นตอนไหนที่ทำให้ model เพี้ยนทั้งไฟล์
   - มีขั้นตอนไหนที่ถ้าพลาดแล้วแก้คืนยาก
   - มีขั้นตอนไหนเสี่ยงสร้างเหล็กซ้อนโดยไม่รู้ตัว

5. Warning coverage
   - จุดไหนควรมี `[!WARNING]`, `[!IMPORTANT]`, หรือ checkpoint แต่ยังไม่มี
   - warning ที่หนังสือมีอยู่ครอบคลุมปัญหาหน้างานจริงพอหรือยัง

รูปแบบผลลัพธ์:

1. `Critical vulnerabilities`
   - รายการความเสี่ยงที่ร้ายแรงที่สุด

2. `Edge-case scenarios`
   - ยกตัวอย่าง what-if scenarios ที่หนังสือยังรองรับไม่พอ

3. `Missing warnings`
   - ระบุชัดว่าควรเพิ่ม warning ที่ประโยคหรือหัวข้อไหน

4. `Idiot-proof score / 10`
   - ให้คะแนนว่าหนังสือกันความผิดพลาดของผู้ใช้ได้ดีแค่ไหน
   - พร้อมเหตุผลสั้นๆ

ข้อห้าม:

- ห้ามวิเคราะห์แบบกว้างๆ
- ห้ามชมรวมโดยไม่ชี้ risk จริง
- ถ้าพบจุดที่พลาดแล้วพังทั้ง workflow ให้รายงานก่อนเรื่องเล็ก
