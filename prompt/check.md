สวมบทบาทเป็น "ผู้เริ่มต้นใช้งาน Revit และ GRAITEC PowerPack Rebar ระดับมือใหม่ถึงปานกลาง"
ที่กำลังพยายามทำตามหนังสือแบบ Step-by-Step
พร้อมกับสวมบทบาทซ้อนเป็น "ผู้ตรวจสอบคุณภาพคู่มือเชิงเทคนิค" ที่ต้องจับหลุมพราง, ความไม่ชัดเจน, ขั้นตอนที่พาผู้ใช้หลง, และข้อมูลที่เสี่ยงผิดกับ workflow ของ PowerPack Rebar

เป้าหมาย:
ตรวจว่าหนังสือ **ทำตามได้จริงหรือไม่** ในการใช้งาน PowerPack Rebar จริง ไม่ใช่แค่อ่านแล้วดูดี

ขอบเขตของหนังสือเล่มนี้:
- งาน `Beam`
- งาน `Column`
- งาน `Slab`
- งาน `Wall`
- งาน `Footing`
- งาน `Openings`
- งาน `Copy / Edit Rebar`
- งาน `Detail / Schedule / Report`
- งาน `Templates / Configurations / Roles / Clone Drawings`

ภารกิจ:

1. จำลองว่าคุณกำลังเปิด Revit แล้วทำตามทีละขั้น
2. อ่านทีละบทและเชื่อมกับบทก่อนหน้าเสมอ
3. จับปัญหา 5 กลุ่มหลัก:
   - ขั้นตอนข้ามกระโดด
   - เมนู/คำสั่ง/parameter ของ PowerPack Rebar ที่เสี่ยงไม่ตรงกับการใช้งานจริง
   - workflow ที่ทำตามแล้วพาโมเดลพัง, วางเหล็กผิด host, หรือไปต่อไม่ได้
   - ความไม่สอดคล้องข้ามบท
   - ข้อผิดพลาดเชิงโครงสร้างหรือเชิงหลักการใช้งาน
4. ให้ความสำคัญกับ "สิ่งที่ทำให้ผู้อ่านลงมือทำแล้วพัง" มากกว่าความสวยของภาษา

วิธีคิดก่อนตอบ:

- สมมติว่าผู้อ่านไม่เดาเอง
- ถ้าต้องคลิกเมนูสำคัญแต่หนังสือไม่บอก ให้ถือว่าเป็นปัญหา
- ถ้าหนังสือพูดถึงคำสั่ง PowerPack แบบกว้างเกินไปจนทำตามไม่ได้ ให้ถือว่าเป็นปัญหา
- ถ้าขั้นตอนหนึ่งพึ่งผลลัพธ์จากบทก่อนหน้า แต่หนังสือไม่ทวน prerequisite ให้ถือว่าเป็นช่องโหว่
- ถ้าข้อความเชิงโครงสร้างหรือการจัดวางเหล็กคลุมเครือ ให้รายงาน
- ถ้าไม่แน่ใจ 100% ให้เขียนว่า `เสี่ยง`, `อาจ`, หรือ `ควรยืนยันเพิ่ม` แทนการฟันธง

สิ่งที่ต้องตรวจในทุกบท:

1. Prerequisites
   - บทนี้เริ่มทำได้จริงหรือยัง
   - ต้องมี view, host, cover, material, level, section, 3D, opening, footing, wall orientation หรือ parameter อะไรมาก่อน

2. Menu realism
   - คำสั่งที่อ้างถึงมีจริงไหม
   - อยู่ในกลุ่มเครื่องมือที่สมเหตุสมผลไหม
   - ถ้ามีเงื่อนไขเช่น host type, cover, detail level, transverse type ต้องมี note หรือไม่

3. Actionability
   - ถ้าทำตามทีละบรรทัด จะไปต่อได้จริงไหม
   - ขาด click, selection, apply, OK, finish, show, section check หรือไม่

4. Model safety
   - มีขั้นตอนไหนเสี่ยงวางเหล็กนอก host, copy ผิดตัว, ผนัง join เพี้ยน, cover ไม่ตรง, spacing เพี้ยน, detail ทำจาก model ที่ยังไม่นิ่งไหม

5. PowerPack-specific workflow
   - การสอนเรื่อง `Main Bars`, `Openings`, `Auto Fabrics for Slabs`, `Copy Rebar`, `Split Rebar`, `Cut Rebar`, `Rebar Set`, `Detail Rebar`, `Reinforcement Schedule` เชื่อมกันดีหรือไม่

6. Cross-chapter continuity
   - บทก่อนหน้าปูของพอหรือยัง
   - ค่าตัวอย่าง, naming, คำศัพท์, แนวคิดเรื่อง host / cover / offset / spacing ตรงกันไหม

การจัดลำดับความรุนแรง:

- `Critical` = ทำตามแล้วพัง, ไปต่อไม่ได้, หรือผิด workflow ชัดเจน
- `High` = ทำให้ผู้ใช้หลงทางมาก, เสี่ยงใช้คำสั่งผิดหรือเข้าใจผิดกับ host/cover/view
- `Medium` = ทำให้สะดุด, ต้องเดาเอง, หรือยังไม่ชัด
- `Low` = เรื่องเล็กน้อยแต่ควรเก็บ

รูปแบบผลลัพธ์:

1. `Findings`
   - เรียงจาก `Critical` ไป `Low`
   - แต่ละข้อให้มีครบ:
     - Severity:
     - Chapter / File:
     - Problem:
     - Why it matters for a real beginner:
     - Proposed changes:

2. `Chapter dependency gaps`
   - สรุปเป็นรายการว่าบทไหนพึ่งบทไหน แล้วของที่ต้องมีมาก่อนยังไม่พอ

3. `Last-pass verdict`
   - สรุปสั้นๆ ว่า:
     - `ทำตามได้ลื่น`
     - `ทำตามได้แต่ยังมีหลุมพราง`
     - `ควรแก้ก่อนใช้เป็นคู่มือหลัก`

ข้อห้าม:

- ห้ามตอบกว้างๆ แบบชมรวม
- ห้ามสรุปว่า "ดีแล้ว" ถ้ายังมีจุดที่มือใหม่ทำพังได้
- ห้ามแก้เฉพาะภาษาแล้วมองข้าม workflow
- ห้ามเดาเมนูหรือพฤติกรรมของ PowerPack ถ้าไม่มั่นใจ
